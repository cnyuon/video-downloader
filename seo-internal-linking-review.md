# SEO Internal Linking Review

Date: 2026-03-17 (America/Los_Angeles)
Scope: Verification of the external crawl report against the local Astro codebase and generated routes.

## 1) Checks and Balances (Data Accuracy)

## Verified Inputs
- External report entries parsed:
  - Orphan list: 236 URLs
  - Missing/Broken list: 135 URLs
- Local verification sources:
  - `frontend/src/pages/**`
  - `frontend/src/content/blog/**`
  - `frontend/astro.config.mjs` redirects
  - Local production build (`npm run build`) route output

## Orphan Claims: Accuracy Summary
- **False positives dominate**.
- Breakdown of the 236 orphan entries:
  - 180 are **mis-mapped blog URLs** (reported as `/blog/{lang}/{slug}` but actual route is `/{lang}/blog/{slug}`).
  - 14 are **components/layout files**, not pages.
  - 11 are **dynamic route source files** (e.g., `[...slug].astro`), not concrete page URLs.
  - 4 are **files not present in this repo** (`ASTRO_SEO_ANALYSIS.md`, `PROJECT_MASTER_PLAN.md`, `i18n_architecture_guide.md`, `notebooklm...md`).
  - 4 are **draft files** under `frontend/drafts`, not published pages.
  - 3 are **non-site project docs** (`backend/deploy/README.md`, `extension/README.md`, `extension/popup.html`).
  - 1 is a public static asset HTML file under `frontend/public/`.
  - 19 are real route files (`/ar`, `/ar/blog`, etc.), but these are not substantiated as truly orphan once templates/footer/sitemap are considered.

## Missing/Broken Claims: Accuracy Summary
- Of 135 reported broken URLs:
  - **110 are false positives** (valid dynamic or redirect-backed routes in local build).
  - **25 are truly unresolved/missing**.

## Important Mismatches Found
1. **Blog URL mapping mismatch** in external crawl data:
   - Report uses `/blog/ar/post-slug`
   - Actual routes are `/{lang}/blog/post-slug`
2. **Dynamic routes were misclassified as missing**:
   - `/blog/{slug}` and `/{lang}/blog/{slug}` are generated by `[...slug].astro` and confirmed in build output.
3. **Stale/out-of-repo references** were included in the external report.
4. **Top PageRank outlink counts are stale/incorrect** relative to current code:
   - Example: built `/tiktok-downloader/` currently renders many internal links (not zero outlinks).

## 6) Final Verdict
- The external crawl is directionally useful but **materially inaccurate** for this codebase without route-aware normalization.
- Most “missing” and “orphan” flags are false positives caused by:
  - wrong locale URL mapping,
  - dynamic route misunderstanding,
  - inclusion of non-page files/out-of-repo files.
- The **real high-priority SEO issues** to fix first are:
  1. Missing `/instagram-downloader` route family,
  2. Missing `best-time-to-post-on-tiktok` route family,
  3. Link distribution skew (underlinked evergreen posts) from high-PR pages.
